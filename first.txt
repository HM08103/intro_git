gitの練習をしてきます。
cd xxx 指定ディレクトリに進む
cd.. 一つ上に戻る
ls ファイル一覧

git init    gitリポジトリを作成。.git隠しフォルダが生成される。
git add     ファイル変更をステージングエリア(控室)へ追加する。
git add xxx 任意ファイルをステージングエリアへ追加する。
git add .   ディレクトリ内すべてのファイルをステージングエリアへ追加する。
git commit  ローカルリポジトリにコミットする。
git commit -v 変更がかかった詳細内容が確認できる。
git push    リモートリポジトリにプッシュする。
git status  ディレクトリ内ファイルの状態を確認する。最後のcommitから更新された分が確認できる
    On branch master:
    origin/master:リモートリポジトリ(githubなど)を指す
git log     更新の履歴が確認できる。commitしきった後のファイル確認できる。
git log --oneline   一行でログを表示する
git log -p xxx      ファイルの差分を表示する
git log -n xxx      指定数文のログだけ確認できる
git rm xxx  指定の削除ファイルをステージングエリアに移行する
git diff    ファイルの変更差分を表示する。ローカルとステージングの差異をみてる。
git diff HEAD       ステージングとコミットの差分を見れる

コミットするファイルを選択するためにあるのがステージエリア
リポジトリに変更を記録することをコミットと呼ぶ。一つの作業ごとにコミットする。
1行目：変更内容の要約
2行目：空行
3行目：変更した理由
